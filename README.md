# NCB Game Data History

Pre-built **ESPN_EVENT** XML files for NCAA Men's Basketball, served as a
public companion dataset for the [Perceptronix NCB Analytics UI](https://ncb-analytics-gateway.poornima2489.workers.dev).

The UI's *Downloads* tab fetches XML files **directly from this repo** (via
`raw.githubusercontent.com`) when the requested games are present, skipping
any database round-trips. Games not yet in the repo automatically fall back
to MotherDuck, then to a live ESPN API fetch — fully transparent to the end
user.

## Layout

```
ncb_xml_2023-24/    6232 files + manifest.json
ncb_xml_2024-25/    6296 files + manifest.json
ncb_xml_2025-26/    6315 files + manifest.json
```

Each `manifest.json` maps `game_id -> filename` for fast random access:

```json
{ "season": "2024-25", "count": 6297, "files": { "401708023": "ncb_20241104_401708023.xml" } }
```

XML filenames follow the legacy convention `ncb_<YYYYMMDD>_<gameId>.xml`
required by Ivan's downstream pipeline.

## Raw URL pattern

```
https://raw.githubusercontent.com/poornimaramakrishnan/ncb_game_data_history/main/ncb_xml_<season>/<filename>
```

## Updating

This dataset is regenerated by the platform repository
(`perceptronix-ncb-data-platform`) using:

```bash
make xml SEASON=2025-26          # rebuild XMLs from cached JSON
python ../setup_history_repo.py  # refresh hardlinks + manifests
git add . && git commit -m "refresh" && git push
```
